Introducción al Entorno de desarrollo integrado (IDE) AVR Studio 7

IDE: AVR Studio

AVR Studio es un Entorno de Desarrollo Integrado (IDE) que permite la gestión de proyectos, la edición de archivos fuente, la simulación, el ensamblaje y front-end para c / c ++, la programación, emulación y depuración en chip.

  • Las distintas versiones del AVR Studio la podemos encontrar desde la página oficial de Microchip:

https://www.microchip.com/mplab/avr-support/avr-and-sam-downloads-archive

IDE AVR Studio 7: Creación de un proyecto

Una vez iniciado el programa AVR Studio aparece como pantalla principal todo lo relacionado a la creación de un nuevo proyecto o bien la apertura de uno ya existente

Fuente: Elaboración propia

  • Para crear un proyecto se selecciona el botón "New Project", entonces aparecerá el siguiente asistente:

    • Fuente: Elaboración propia

  • Aquí, se seleccionará el tipo de lenguaje a utilizar para desarrollar el proyecto, ya sea en Assembler o C/C++.

    • Fuente: Elaboración propia

    En cual sea de los casos, se debe indicar el nombre y la ubicación donde éste será guardado (para este primer ejemplo empezaremos con crear un proyecto en Assembler)

  • Una vez definido el tipo de proyecto a trabajar y la ubicación del mismo, se procedemos a seleccionar el dispositivo a utilizar, que, en estos casos, utilizaremos el ATmega328P

    • Fuente: Elaboración propia

Código Assembler de ejemplo

Para adentrarnos un poco partiremos del siguiente ejemplo realizado en Assembler el cual copiaremos y pegaremos en el entorno AVR Studio (creando un archivo .asm en primera instancia)

copiar código
.INCLUDE "M328PDEF.INC"
.DEF DATOA=R16
.DEF DATOB=R17
.DEF CODIGO=R18
.DEF RESULT=R21

.ORG 0 ; ALMACENARA EN LA LINE O DE PROGRAM MEMORY
.CSEG ; INICIO DEL SEGMENTO DE CODIGO DE PROGRAMA
RJMP INICIO

INICIO:
LDI R20
, 255
OUT PORTB, R20 ; PULL UP EN PORTB
OUT PORTC, R20 ; PULL UP EN PORTC
OUT DDRD, R20 ; PORTD SALIDA
LDI R19, 0
OUT DDRB, R19 ; PORTB ENTRADA
OUT DDRC, R19 ; PORTC ENTRADA
LEER:
IN DATOA, PINC ;LEE PORTC Y LO GUARDA EN DATOA
ANDI DATOA, $07 ;ENMASCARA DATOA
IN DATOB, PINC ;LEE PORTC OTRA VEZ
ANDI DATOB, $38 ; DATOC ESTA EN LA PARTE ALTA. SE DEBE DESPLAZAR 3 POSICIONES A LA DERECHA
LSR DATOB
LSR DATOB
LSR DATOB
IN CODIGO, PINB ;LEE PORTC
ANDI CODIGO, $07 ; ENMASCARA LOS 3LSB
CASE:
CPI CODIGO, 0
BREQ COD0
CPI CODIGO, 1
BREQ COD1
CPI CODIGO, 2
BREQ COD2
CPI CODIGO, 3
BREQ COD3
CPI CODIGO, 4
BREQ COD4
CPI CODIGO, 5
BREQ COD5
CPI CODIGO, 6
BREQ COD6
CPI CODIGO, 7
BREQ COD7
MOSTRAR:
OUT PORTD, RESULT
RJMP LEER
COD0:
ADD DATOA, DATOB
MOV RESULT, DATOA
RJMP MOSTRAR
COD1:
SUB DATOA, DATOB
MOV RESULT, DATOA
RJMP MOSTRAR
COD2:
MUL DATOA, DATOB
MOV RESULT, R0
RJMP MOSTRAR
COD3:
AND DATOA, DATOB
MOV RESULT, DATOA
RJMP MOSTRAR
COD4:
OR DATOA, DATOB
MOV RESULT, DATOA
RJMP MOSTRAR
COD5:
EOR DATOA, DATOB
MOV RESULT, DATOA
RJMP MOSTRAR
COD6:
COM DATOA
MOV RESULT, DATOA
RJMP MOSTRAR
COD7:
COM DATOB
MOV RESULT, DATOA
RJMP MOSTRAR

Observemos pues, que al pegar el programa, este cambia de colores. Las instrucciones se observan azules, los comentarios en color verde, etc. Así se puede identificar claramente cada elemento de una línea de programa.

Fuente: Elaboración propia

Compilación del código fuente

Una vez realizado el programa escrito hay que ensamblar el código, esto es llevar de código fuente a código máquina. En el entorno de AVR Studio, se le llama construir (Build).

El ensamblaje se realiza en menú Build, opción Build Solution (F7). También puede utilizar el icono de la barra de herramientas

Fuente: Elaboración propia

En caso de producirse errores, en la ventana de mensajes aparecerá el número de línea del programa donde está el error

Fuente: Elaboración propia

  • Haciendo doble click sobre el error, el cursor se posicionará sobre la línea que presenta el error
  • Al corregirlo se deberá volver a compilar, esto se debe realizar tantas veces sean necesarias para no tener errores en el programa

Si todo es correcto debe aparecer en la ventana de mensajes la información referente al programa compilado

Fuente: Elaboración propia

Una vez compilado el programa sin errores, se generarán, dentro de la carpeta del proyecto, diversos archivos como el .hex y el .obj

Analizando el código: Uso del Datasheet ATmega328P

Analicemos pues, el funcionamiento del código escrito partiendo de las instrucciones escritas y el Set de instrucciones dadas por el datasheet del Atmega328P

  • Partiremos desde este segmento del código:

    • Fuente: Elaboración propia

  • Esto nos indica que la instrucción LDI requiere de dos operandos para funcionar (Rd, k), donde k es un valor constante entre 0 y 255 ($00 a $FF en hexadecimal), cuya función es cargar el valor k al registro Rd (Rd <- K)
  • Partiendo del segmento del código escogido LDI nos indica que se guarda el valor de 255 ($FF) en el registro R20

    • Fuente: Elaboración propia

  • Esto nos indica que lo que esté guardado en el registro Rr se guardará en el registro P (P <- Rr)
  • Quiere decir que el valor anteriormente guardado en el registro R20 (255) configurará el registro del puerto B (PORTB) del microcontrolador, así como también el registro del puerto C (PORTC) y la directriz del puerto D (DDRD) colocándole a todos estos el valor 255 ($FF en hexadecimal)

Simulando el programa dentro del IDE AVR Studio 7

AVR Studio incluye un simulador con el cual se puede visualizar el estado del microcontrolador durante la ejecución de un programa

Entre las opciones más usadas están:

  • Ejecución paso a paso
  • Ejecución hasta un punto de parada (Breackpoint).

Para comenzar la simulación del programa accedemos al menú Debug y utilizamos la opción "Start Debugging and break".

Fuente: Elaboración propia

Tras entrar en el modo depuración observamos lo siguiente:

Fuente: Elaboración propia

Barra de simulación:

Fuente: Elaboración propia

Cómo añadir un Breackpoint:

Para añadir un nuevo “Breakpoint” ubicamos el cursor sobre la línea de código donde se establecerá el breakpoint y, sobre la columna gris (donde también se muestra la siguiente instrucción a ejecutar) damos doble click, a lo que, se establecerá el nuevo breakpoint, de esta manera:

Fuente: Elaboración propia

Para comenzar la simulación, efectuamos inicialmente un Reset, ya que esto simula lo que sucede cuando se energiza el microcontrolador

El Program Counter es cargado con la dirección del Reset Vector (00h), lo que justifica que el origen de la mayoría de los programas sea justamente esta posición en la memoria de programa

Fuente: Elaboración propia

A continuación presionamos varias veces el botón Step Into o Step Over y observamos que le ocurre a los registros. Observemos también, que la flecha amarilla apunta a la próxima instrucción a ejecutar.

Fuente: Elaboración propia

  • Se ilustrará la manera de modificar el valor de los registros, lo cual es útil cuando el flujo del programa depende de la ocurrencia de eventos específicos que tardarían mucho si se sigue una simulación paso a paso, como por ejemplo la ejecución de ciclos de retardo de tiempo.
  • Para cambiar el valor de un registro simplemente hacemos doble clic en él y asignamos el nuevo valor
