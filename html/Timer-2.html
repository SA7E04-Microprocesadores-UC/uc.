Timer 2 en el ATmega328P

Timer2 en el ATmega328P:

Fundamentos

A diferencia del Timer0 y el Timer1 el Timer2 no funciona como contador, solo como temporizador y puede ser usado en modo comparador, y también en modo PWM, para utilizarse el timer2 como temporizador, este tiene dos opciones, el timer2 puede trabajar con el mismo cristal o reloj con el cual trabaja el microcontrolador, o el timer2 puede trabajar con un cristal o reloj independiente conectado exteriormente a los pines TOSC1 y TOSC2 (Pin PB5 y PB6 del ATmega328P respectivamente), en este caso se dice que trabaja en forma asíncrona al sistema

Diagrama de tiempo

El timer2 es un temporizador de 8 bits que cuenta con el registro TCNT2 para almacenar sus valores de temporización que aumentara de uno en uno en cada ciclo de reloj, ya sea el reloj del sistema o el reloj externo

Fuente: Microcontrollers ATmega328P Datasheet

En el diagrama anterior podemos apreciar como va incrementando el registro TCNT2. En esta ocasión, se presenta un diagrama de tiempo sin pre escalador (𝑐𝑙𝑘𝐼/𝑂/1) es decir, el registro TCNT2 se incrementará por cada ciclo (𝑇) de reloj del microcontrolador (𝑐𝑙𝑘𝐼/𝑂).

Timer2 como temporizador (con reloj del sistema):

Configuración del Registro TCCR2B

Para programar el Timer2 como temporizador se cuenta con el registro TCCR2B que nos permitirá configurarlo de esta manera

En primera instancia, el timer2 cuenta con lo que se conoce como el pre escalador, esto hace que la frecuencia de trabajo (𝐹𝐶𝑃𝑈) se divida por este pre escalador, con lo que se logra que el timer2 tarde un poco mas en aumentar su valor en una unidad; el pre escalador puede tomar el valor de 1, 8, 32, 64, 256 o 1024;estos valores se eligen programando los bits 0, 1 y 2 del registro TCCR2B, los bits 7 a 3 en este caso se pondrán a 0.

Ejemplo: Uso del Pre escalador

Si el pre escalador utilizado es, por ejemplo, de 256 y la 𝐹𝐶𝑃𝑈 = 16𝑀𝐻𝑧, entonces la frecuencia de trabajo del timer2 𝐹𝑇𝑖𝑚𝑒𝑟2 será de 𝐹𝑇𝑖𝑚𝑒𝑟2 = 𝐹𝐶𝑃𝑈 ∕𝑃𝑟𝑒𝑒𝑠𝑐𝑎𝑙𝑎𝑑𝑜𝑟, el tiempo que tardará ahora el temporizador en aumentar una unidad será la inversa de este valor lo cual se conoce como periodo: 𝑇𝑇𝑖𝑚𝑒𝑟2 = 𝑃𝑟𝑒𝑒𝑠𝑐𝑎𝑙𝑎𝑑𝑜𝑟∕𝐹𝐶𝑃𝑈 por lo que en este caso 𝑇𝑇𝑖𝑚𝑒𝑟2 = 256∕16𝑀𝐻𝑧 donde 𝑇𝑇𝑖𝑚𝑒𝑟2 = 16𝑢𝑠, esto quiere decir que ahora el timer2 aumentará en una unidad cada 16𝑢𝑠 y, en este caso, el registro TCNT2 para ir de 0 a 255 tardará 255∗𝑇𝑇𝑖𝑚𝑒𝑟2 = 255∗16𝑢𝑠 = 4.08 𝑚𝑠; y así con la misma idea para los demás pre escaladores.

Tiempo Máximo del Timer2

De acuerdo a los tiempos que se quieran obtener al utilizar el timer2, habrá que utilizar el pre escalador adecuado así como inicializar el registro TCNT2 también con un valor adecuado, no olvidemos que este registro es de 8 bits y por lo tanto solo puede contener valores enteros entre 0 y 255

Cabe destacar, en primera instancia, que para establecer un determinado tiempo del temporizador hay que considerar primero cuanto es el máximo tiempo que puede generar el mismo, para así, evitar inconvenientes al momento de establecer el valor inicial del registro TCNT2

Ejemplo 1: Timer2 como temporizador

En este primer ejemplo utilizaremos los cálculos obtenidos anteriormente para apagar y encender un LED conectado al bit 0 del puerto D (PD0) cada 15 ms el cual será visualizado por medio de un osciloscopio

copiar código
#define F_CPU 16000000UL // Frecuencia de trabajo del microcontrolador
#include <avr/io.h>

int main(void) {
DDRD = (1 << 0); // bit 0 del PuertoD como salida
PORTD = ~(1 << 0); // bit inicialmente apagado
TCCR2B = 0b00000111; // Preescaler de 1024
TCNT2 = 20; // Valor inicial

while (1) {
if (TCNT2 == 255) {
PORTD ^= (1 << 0); //Cuando TCNT2 llegue a su valor Max. se invierte el estado del bit
TCNT2 = 20; // se vuelve a cargar el valor inicial para tener la misma base de tiempo
}
}
}

Ejemplo 2: Timer2 como temporizador con base de tiempo más prolongada

En este segundo ejemplo se muestra como se pueden obtener bases de tiempos mucho mayores al tiempo máximo del timer2, basta con incorporar una variable tipo contador y establecer una base fija de tiempo, con los cálculos anteriores se obtuvo una base de 15ms por lo que, para establecer un retardo de 3 segundos la variable tipo contador tiene que llegar a un total de 200 unidades (15 𝑚𝑠 ∗ 200 = 3 𝑠𝑒𝑔𝑢𝑛𝑑𝑜𝑠)

copiar código
#define F_CPU 16000000UL // Frecuencia de trabajo del microcontrolador
#include <avr/io.h>
#include <util/delay.h>

uint8_t Count = 0; // Variable tipo contador

int main(void) {
DDRD = (1 << 0); // bit 0 del PuertoD como salida
PORTD = ~(1 << 0); // Led inicialmente apagado
TCCR2B = 0b00000111; // Preescaler de 1024
TCNT2 = 20; // Valor inicial

while (1) {
if (TCNT2 == 255) {
Count++; // la variable incrementa cada 15 ms
TCNT2 = 20; // se vuelve a cargar el valor inicial para tener la misma base de tiempo
}

if (Count == 200) {
PORTD ^= (1 << 0); //Cuando la Variable Count llegue a 200 ha transcurrido aprox. 3 segundos y se invierte el estado del bit0
Count = 0; // vuelve a iniciar el conteo de 1 segundo
_delay_ms(10);
}
}
}

Timer2 modo CTC:

Al utilizar el timer2 en modo comparador ocurre que el registro TCNT2, que es de 8 bits, irá aumentando su valor de uno en uno desde 0 hasta que este se haga igual a un valor almacenado en el registro OCR2A, en el momento que se hacen iguales los valores de los registros TCNT2 y OCR2A, el registro TCNT2 se pone automáticamente a cero o también se dice que se borra, para nuevamente comenzar a aumentar su valor desde 0 hasta que se vuelva hacer igual al valor almacenado en el registro OCR2A

Al mismo tiempo una bandera de comparación es activada (OCF2A o OCF2B según sea el caso) los que nos permitirá manejar interrupciones

Fuente: Elaboración propia

A esto se lo conoce como el modo de trabajo CTC, que quiere decir borrado del timer por comparación

Registros involucrados

En el modo de trabajo CTC del timer2 se utilizarán los registros TCCR2A y TCCR2B

Configuración del Registro TCCR2A

Con este registro se elige si el evento a ocurrir como respuesta por comparación a la igualdad entre el registro TCNT2 y el registro OCR2A, será por el pin OC2A u OC2B

Los bits 7 y 6 controlan el comportamiento del pin de comparación de salida (OC2A o bien el pin PB3 del ATmega328P), para elegir el tipo de evento que ocurrirá cuando se iguale el registro TCNT2 con el registro OCR2A, en el caso de utilizar el pin OC2A para las combinaciones de los bits 7 y 6 se tiene la siguiente tabla, si ambos bits son 0 la opción OC2A del pin no está disponible por lo que el pin se comporta como una entrada/salida digital normal

Fuente: Microcontrollers ATmega328P Datasheet

Si por ejemplo el bit7 se pone a 0 y el bit6 a 1 entonces el estado del pin OC2A cambiará cada vez que la comparación ocurra,si el bit7 y el bit6 se ponen a 1 ocurrirá que si el pin OC2A estaba a 0 este se pondrá a 1, pero si estaba a 1 este permanecerá igual

Fuente: Microcontrollers ATmega328P Datasheet

Estos bits controlan el comportamiento del pin de comparación de salida (OC2B o bien el pin PD3 del ATmega328P) para elegir el tipo de evento que ocurrirá cuando se iguale el registro TCNT2 con el registro OCR2A, en el caso de utilizar el pin OC2B para las combinaciones de los bits 5 y 4 se tiene la siguiente tabla, si ambos bits son 0 la opción OC2B del pin no está disponible por lo que el pin se comporta como una entrada/salida digital normal

Fuente: Microcontrollers ATmega328P Datasheet

Si por ejemplo el bit5 se pone a 0 y el bit4 a 1 entonces el estado del pin OC2B cambiará cada vez que la comparación ocurra, si el bit5 y el bit4 se ponen a 1 ocurrirá que si el pin OC2B estaba a 0 este se pondrá a 1, pero si estaba a 1 este permanecerá igual

Los bits 3 y 2 no se utilizan por lo que se les suele poner a 0

El bit1 para utilizar el modo CTC tiene que ser puesto a 1 y el bit0 puesto a 0

Configuración del Registro TCCR2B

Con este registro se elige el pre escalador a utilizar, para obtener en cuanto tiempo se quiere que el registro TCNT2 se iguale al registro OCR2A

Configuración del registro OCR2A

En el registro OCR2A se cargará el valor hasta el cual se quiere que llegue el registro TCNT2 en el modo CTC, por lo que hay que obtener el valor adecuado a cargar en el registro OCR2A, esto se realiza mediante la siguiente expresión:

El registros OCR2A es de 8 bits por lo que el valor obtenido mediante la ecuación anterior tendrá que ser un número entero comprendido entre 0 y 255, de no ser así el valor no será válido

Por ejemplo, para lograr que la comparación ocurra cada 5ms, utilizando un pre escalador de 1024 será necesario cargar el registro OCR2A con el valor de:

78 sería el valor a cargar en el registro OCR2A para que cada 5ms el valor del registro TCNT2 llegue a 78 y por comparación se produzca un evento en el pin OC2A u OC2B, tras lo cual el registro TCNT2 se pondrá automáticamente a 0 para volver a aumentar su valor nuevamente hasta 78 en un lapso de 5ms y se vuelva ha repetir el proceso

Ejemplo: Uso del Timer2 como comparador

Para el ejemplo del uso del temporizador contador timer2 en modo CTC, se utilizará el pin PD3 o bien el OC2B del ATmega328P como el pin en el que ocurrirá un evento, que será cada vez que ocurra la igualdad entre el registro TCNT2 y el registro OCR2A donde el estado de este pin OC2B cambie, este pin hay que configurarlo como una salida digital mediante el DDRD correspondiente, se hará que el pin OC2B cambie de estado cada 5ms, esto será observado mediante un osciloscopio conectado a este pin

Utilizando los cálculos anteriores,cargamos el registro OCR2A = 78, en el registro TCCR2A hay que poner a 1 su bit4 para que el evento por cambio de estado sea por el pin OC2B, y también hay que poner a 1 su bit1 para hacer trabajar el timer2 el modo CTC, sus demás bits se ponen a 0

TCCR2A = 0b00010010; //cambio estado en el pin OC2B modo CTC

Del registro TCCR2B se colocará a 1 su bit2, a 1 el bit1 y a 1 el bit0 para tener el pre escalador de 1024, para hacer que un led conectado al pin OC2B cambie de estado cada 5ms

TCCR2B = 0b00000111; //prescaler de 1024

El código completo queda de la siguiente manera:

copiar código
#define F_CPU 16000000UL // Frecuencia de trabajo del microcontrolador
#include <avr/io.h>

int main(void) {
DDRD = (1 << 3); // bit 3 del PuertoD como salida
PORTD = ~(1 << 3); // bit inicialmente apagado
TCCR2A = 0b00010010; //cambio estado en el pin OC2B modo CTC
TCCR2B = 0b00000111; //prescaler de 1024
OCR2A = 78; // Cuando el registro TCNT2 alcance este valor han transcurrido 5ms y luego se inicializara a 0 para empezar un nuevo ciclo
while (1) {}
}
