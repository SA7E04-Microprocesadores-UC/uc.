PWM Timer0 Modo Fase Correcta

PWM Modo: Fase correcta

El modo PWM de fase correcta proporciona una generación de forma de onda PWM de fase correcta de alta resolución. El modo PWM de fase correcta se basa en una operación de doble pendiente. El contador cuenta repetidamente desde abajo (Bottom) hasta arriba (Top) y luego de arriba (Top) hasta abajo (Bottom)

Fuente: Elaboración propia

Aquí, el contador (el registro TCNT0) irá desde 0 (BOTTOM) hasta 255 (TOP; por ser un registro de 8 bits), y luego de 255 a 0 y así continuará en el modo fase correcta; a este ir de 0 a 255 y de 255 a 0 le tomará un tiempo que dependerá del pre escalador utilizado para el timer0, ese tiempo que le tome al registro TCNT0 para ir de 0 a 255 y de 255 a 0 es el periodo de la señal PWM fase correcta, al registro TCNT0 se lo puede imaginar como si estuviese generando una onda triangular a lo largo del transcurso del tiempo

Debido a la característica simétrica de los modos PWM de doble pendiente, estos modos son los preferidos para aplicaciones basadas en control de motores

Obtención de la señal PWM

La señal PWM se genera partiendo de la configuración del timer0 en modo comparación o modo CTC, esta se genera cuando el valor del  registro TCNT0, el que se encuentra incrementándose de unidad en unidad, se iguala al valor almacenado en el registro OCR0A o en el registro OCR0B, tanto cuando el registro se incrementa de 0 a 255 como cuando se decrementa de 255 a 0. Cuando se utilizó el timer0 en modo comparación al ocurrir esta igualdad el registro TCNT0 se reiniciaba a 0, en el modo PWM el registro TCNT0 no se reiniciará tras ocurrir la igualdad sino que seguirá incrementado sus valores hasta llegar a su máximo valor de 255 o decrementandose hasta su valor mínimo de 0.

La señal PWM generada se obtendrá en cualesquiera de los pines OC0A o OC0B del ATmega328P, dependiendo si para la comparación se utiliza el registro OCR0A o el registro OCR0B, a partir de este momento se utilizará OC0x para los pines y OCR0x para los registros de comparación donde x puede ser A o B

Una cosa a tener en cuenta es que al ocurrir la igualdad entre el registro TCNT0 y el registro OCR0A, se puede habilitar el uso de la interrupción por comparación del timer0.

Obtención de la señal PWM: No invertida

En el modo PWM fase correcta, la unidad de comparación permite la generación de formas de onda PWM en los pines OC0x. Esta puede ser en forma invertida o en forma no invertida, estos pines deben ser declarados como salidas digitales mediante sus registros DDRx respectivos

Fuente: Elaboración propia

Para la forma no invertida por el pin OC0x se obtendrá un alto mientras el valor del registro TCNT0 este aumentando de 0 a 255 y sea menor al valor almacenado en el registro OCR0x, al ocurrir la igualdad entre los registros TCNT0 y OCR0x el estado del pin OC0x cambiará a un bajo y se mantendrá así hasta que cuando el registro TCNT0, al disminuir de 255 a 0, se haga igual nuevamente al valor almacenado en registro OCR0x, momento en el cual el estado del pin OC0x cambiará nuevamente a un alto; el registro TCNT0 seguirá disminuyendo hacia 0, luego volverá a incrementase hacia 255 y cuando nuevamente se haga igual al valor almacenado en el registro OCR0x nuevamente por el pin OC0x se obtendrá un bajo y el ciclo se repetirá

Obtención de la señal PWM: Invertida

Fuente: Elaboración propia

Para la forma invertida por el pin OC0x se obtendrá un bajo mientras el valor del registro TCNT0 este aumentando de 0 a 255 y sea menor al valor almacenado en el registro OCR0x, al ocurrir la igualdad entre los registros TCNT0 y OCR0x el estado del pin OC0x cambiará a un alto y se mantendrá así hasta que cuando el registro TCNT0, al disminuir de 255 a 0 se haga igual nuevamente al valor almacenado en registro OCR0x, momento en el cual el estado del pin OC0x cambiará nuevamente a un bajo, el registro TCNT0 seguirá disminuyendo hacia 0, luego volverá a incrementase hacia 255 y cuando nuevamente se haga igual al valor almacenado en el registro OCR0x nuevamente por el pin OC0x se obtendrá un alto y el ciclo se repetirá.

Obtención de la señal PWM: Modificación del ancho de pulso

Para modificar el ancho de pulso de la señal PWM fase correcta solo hay que modificar el valor almacenado en el registro OCR0x, con esto se logra que los cambios de estado del pin OC0x sean en diferentes tiempos, por ejemplo si se toma como referencia la imagen anterior para el nivel del registro OCR0x, cambiando su nivel se obtendrá la señal PWM con un ancho de pulso modificado pero manteniendo constante su periodo, como se muestra a continuación

Para programar el módulo PWM modo rápido con el timer0, se utilizan 2 registros los que son: El registro TCCR0A y el registro TCCR0B

Registro OCR0A/OCR0B

El ancho de pulso en una señal PWM fase correcta no invertido, se inicia cuando el registro TCNT0, que esta incrementándose desde 0 a 255, se iguala al valor almacenado en el registro OCR0x, y terminará cuando al decrementarse el registro TCNT0, que va de 255 a 0, se vuelva a igualar al valor almacenado en el registro OCR0x, por lo que el ancho de pulso será igual a 2 veces el valor almacenado en el registro OCR0x, el periodo de la señal PWM fase correcta es 2 veces el valor máximo que puede tener el registro TCNT0, esto es, 510, ya que va de 0 a 255 y luego vuelve de 255 a 0

Configuración del registro TCCR0A y TCCR0B

Registro TCCR0A

Como se obtendrá la señal PWM por el pin OC0B su bit5 se pondrá a 1 y su bit4 se pondrá a 0, su bit1 se pondrá a 0 y su bit0 se pondrá a 1, ya que se utilizará el modo PWM fase correcta, sus demás bits se pondrán a 0 ya que no se usarán

TCCR0A = 0b00100001; //PWM por el pin OC0B, PWM fase correcta
Registro TCCR0B

Su bit3 se pondrá a 0 para utilizar el modo PWM fase correcta, y como se usará un pre escalador de 64 la combinación de sus bits 2, 1 y 0 será 011, todos sus demás bits se pondrán a 0 ya que no se usarán

TCCR0B = 0b00000011; //PWM fase correcta, prescaler de 64

Código en C y simulación en Proteus

copiar código
#define F_CPU 16000000UL // Frecuencia de trabajo del microcontrolador
#include <avr/io.h>

int main(void) {
DDRD = (1 << 5); // pin OC0B como salida
TCCR0A = 0b00100001; //PWM por el pin OC0B, PWM fase correcta
TCCR0B = 0b00000011; //PWM modo fase correcta, prescaler de 64
OCR0B = 77; // Cuando el registro TCNT0 alcance este valor han transcurrido 10ms y luego se inicializara a 0 para empezar un nuevo ciclo
while (1) {}
}

Ejemplo 2: Ancho de pulso variable

En este ejemplo se generará una señal PWM con un ancho de pulso variable desde el 0% hasta el 100% por el pin OC0A en forma no invertida. Se utilizará el ATmega328P el cual estará trabajando a una frecuencia de 16Mhz

Frecuencia y Registro OCR0A

Procediendo de la misma manera que en el ejemplo 1, se tiene una frecuencia igual a:

Se iniciará el ancho de pulso a un 0% por lo que el registro OCR0A se cargará en un inicio con 0, luego dentro del ciclo del programa se hará que se incremente en una unidad cada 100 ms

OCR0A = 0; //Ancho de pulso inicial de 0

Configuración del registro TCCR0A y TCCR0B

Registro TCCR0A

Como se obtendrá la señal PWM por el pin OC0B su bit5 se pondrá a 1 y su bit4 se pondrá a 0, su bit1 se pondrá a 0 y su bit0 se pondrá a 1, ya que se utilizará el modo PWM fase correcta, sus demás bits se pondrán a 0 ya que no se usarán

TCCR0A = 10000001; //PWM por el pin OC0A, PWM fase correcta
Registro TCCR0B

Su bit3 se pondrá a 0 para utilizar el modo PWM fase correcta, y como se usará un pre escalador de 64 la combinación de sus bits 2, 1 y 0 será 011, todos sus demás bits se pondrán a 0 ya que no se usarán

TCCR0B = 0b00000011; //PWM fase correcta, prescaler de 64

Código en C y simulación en Proteus

copiar código
#define F_CPU 16000000UL // Frecuencia de trabajo del microcontrolador
#include <avr/io.h>
#include <util/delay.h>

int main(void) {
DDRD = (1 << 6); // pin OC0A como salida
TCCR0A = 0b10000001; //PWM por el pin OC0A, PWM fase correcta
TCCR0B = 0b00000011; //PWM modo fase correcta, prescaler de 64
OCR0A = 0; //inicio del conteo

while (1) {
OCR0A += 1; //el ancho de pulso se incrementa en una unidad cada 100 ms
_delay_ms(100);
}
}

Ejemplo 3: Manipulación del ancho de pulso

En este ejemplo se generará una señal PWM con un ancho de pulso variable desde el 0% hasta el 100% por el pin OC0A en forma no invertida en pasos de 25%, es el mismo ejemplo que el caso anterior pero esta vez se utilizará un pulsador conectado al pin PB5 para incrementar el valor del ancho de pulso y un pulsador conectado al pin PB4 para disminuirlo.

Fuente: Elaboración propia

Código en C y simulación en Proteus

copiar código
#define F_CPU 16000000UL // Frecuencia de trabajo del microcontrolador
#include <avr/io.h>
#include <util/delay.h>

int main(void) {
DDRD = (1 << 6); // pin OC0A como salida
TCCR0A = 0b10000001; //PWM por el pin OC0A, PWM fase correcta
TCCR0B = 0b00000011; //PWM modo fase correcta, prescaler de 64
OCR0A = 0; //inicio del conteo

while (1) {
if (!(PINB & (1 << PORTB5))){ //si se presiona el pulsador
_delay_ms(150); // retardo antirebote
OCR0A += 63; //el ancho de pulso se incrementa un 25%
_delay_ms(100);
}
if (!(PINB & (1 << PORTB4))){ //si se presiona el pulsador
_delay_ms(150); // retardo antirebote
OCR0A -= 63; //el ancho de pulso se decrementa un 25%
_delay_ms(100);
}
}
}
