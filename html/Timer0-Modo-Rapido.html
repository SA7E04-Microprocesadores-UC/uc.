PWM Timer0 Modo rápido

PWM Modo: Rápido

La modulación de ancho de pulso rápido o el modo PWM rápido proporciona una opción de generación de una forma onda PWM de alta frecuencia. El PWM rápido se diferencia de la otra opción PWM por su operación de pendiente única.

Fuente: Elaboración propia

Aquí, el contador (el registro TCNT0) irá desde 0 (BOTTOM) hasta 255 (TOP; al ser un registro de 8 bits), luego se reiniciará a 0 para volver hasta 255 y luego, otra vez, volverá a contar de 0 a 255 y así continuará en el modo rápido; el tiempo que toma en ir de 0 a 255 dependerá del pre escalador utilizado para el timer0, y ese tiempo que le tome al registro TCNT0 para ir de 0 a 255 será el periodo de la señal PWM, al registro TCNT0 se lo puede imaginar como si estuviese generando una onda diente de sierra en el transcurso del tiempo.

Debido a la operación de pendiente única, la frecuencia de operación del modo PWM rápido puede ser dos veces más alta que la modo PWM de fase correcta que utiliza la operación de doble pendiente. Esta alta frecuencia hace que el modo PWM rápido sea adecuado para Regulación de potencia, rectificación y aplicaciones DAC. La alta frecuencia permite componentes externos físicamente pequeños (bobinas, condensadores) y, por lo tanto, reduce el costo total del sistema.

Obtención de la señal PWM

La señal PWM se genera partiendo de la configuración del timer0 en modo comparación o modo CTC, esta se genera cuando el valor del  registro TCNT0, el que se encuentra incrementándose de unidad en unidad, se iguala al valor almacenado en el registro OCR0A o en el registro OCR0B, cuando se utilizó el timer0 en modo comparación al ocurrir esta  igualdad el registro TCNT0 se reiniciaba a 0, en el modo PWM el registro TCNT0 no se reiniciará esta vez tras ocurrir la igualdad, sino que, seguirá incrementado sus valores hasta llegar a su máximo valor de 255, momento en el cual se reiniciará.

La señal PWM generada se obtendrá en cualesquiera de los pines OC0A o OC0B del ATmega328P, dependiendo si para la comparación se utiliza el registro OCR0A o el registro OCR0B, a partir de este momento se utilizará OC0x para los pines y OCR0x para los registros de comparación donde x puede ser A o B

El indicador de desbordamiento del temporizador/contador (TOV0) se activa cada vez que el contador llega a su máximo valor (TOP). Si la interrupción está habilitada, La rutina de interrupción se puede utilizar para actualizar el valor de comparación

Obtención de la señal PWM: No invertida

En el modo PWM rápido, la unidad de comparación permite la generación de formas de onda PWM en los pines OC0x. Esta puede ser en forma invertida o en forma no invertida, estos pines deben ser declarados como salidas digitales mediante sus registros DDRx respectivos

Fuente: Elaboración propia

Para la forma no invertida por el pin OC0x se obtendrá un alto mientras el valor del registro TCNT0 sea menor al valor almacenado en el registro OCR0x, al ocurrir la igualdad entre los registros TCNT0 y OCR0x el estado del pin OC0x cambiará a un bajo y se mantendrá así hasta que el registro TCNT0 llegue a su máximo que es 255 y vuelva 0, momento en el cual el estado del pin OC0x cambiará nuevamente a un alto (dado que en este momento el valor del registro TCNT0 pasará a ser menor que el valor del registro OCR0x) y el ciclo se repetirá.

Obtención de la señal PWM: Invertida

Fuente: Elaboración propia

Para la forma invertida por el pin OC0x se obtendrá un bajo mientras el valor del registro TCNT0 sea menor al valor almacenado en el registro OCR0x, al ocurrir la igualdad entre los registros TCNT0 y OCR0x el estado del pin OC0x cambiará a un alto y se mantendrá así hasta que el registro TCNT0 llegue a su máximo que es 255 y vuelva a 0, momento en el cual el estado del pin OC0x cambiará nuevamente a un bajo y el ciclo se repetirá

Obtención de la señal PWM: Modificación del ancho de pulso

Para modificar el ancho de pulso de la señal PWM solo hay que modificar el valor almacenado en el registro OCR0x, con esto se logra que los cambios de estado del pin OC0x sean en diferentes tiempos, por ejemplo si se toma como referencia la imagen anterior para el nivel del registro OCR0x, cambiando su nivel se obtendrá la señal PWM con un ancho de pulso modificado pero manteniendo constante su periodo, como se muestra en la imagen

Configuración del registro TCCR0A y TCCR0B

Registro TCCR0A

Como se obtendrá la señal PWM por el pin OC0B su bit5 se pondrá a 1 y su bit4 se pondrá a 0, sus bits 1 y 0 se pondrán a 1 ya que se utilizará el modo PWM rápido, sus demás bits se pondrán a 0 ya que no se usarán

TCCR0A = 0b00100011; //PWM por el pin OC0B, PWM modo rápido
Registro TCCR0B

Su bit3 se pondrá a 0 para utilizar el modo rápido, y como se usará un pre escalador de 64 la combinación de sus bits 2, 1 y 0 será 011, todos sus demás bits se pondrán a 0 ya que no se usarán

TCCR0B = 0b00000011; //PWM modo rapido, prescaler de 64

Ejemplo 2: Ancho de pulso variable

En este ejemplo se generará una señal PWM con un ancho de pulso variable desde el 0% hasta el 100% por el pin OC0A en forma no invertida. Se utilizará el ATmega328P el cual estará trabajando a una frecuencia de 16Mhz

Fuente: Elaboración propia

Frecuencia y Registro OCR0A

Procediendo de la misma manera que en el ejemplo 1, se tiene una frecuencia igual a:

Se iniciará el ancho de pulso a un 0% por lo que el registro OCR0A se cargará en un inicio con 0, luego dentro del ciclo del programa se hará que se incremente en una unidad cada 100 ms

OCR0A = 0; //Ancho de pulso inicial de 0

El registro OCR0A irá, por lo tanto, desde 0 para el 0% hasta 255 para el 100% del ancho de pulso, momento en que el ancho de pulso se iguala al periodo de la señal PWM; cuando el registro OCR0A llega 255 se reinicia automáticamente a 0, volviendo nuevamente a incrementarse en 1 unidad dentro del ciclo del programa cada 100ms y así continuará, en este ejemplo lo que interesa es ver como se logra la variación del ancho de pulso de una señal PWM en modo rápido

Configuración del registro TCCR0A y TCCR0B

Registro TCCR0A

Como se obtendrá la señal PWM por el pin OC0A su bit7 se pondrá a 1 y su bit6 se pondrá a 0, sus bits 1 y 0 se pondrán a 1 ya que se utilizará el modo PWM rápido, sus demás bits se pondrán a 0 ya que no se usarán

TCCR0A = 0b10000011; //PWM por el pin OC0A, no invertido, PWM modo rápido
Registro TCCR0B

Su bit3 se pondrá a 0 para utilizar el modo PWM rápido, y como se usará un pre escalador de 64 la combinación de sus bits 2, 1 y 0 será 011, todos sus demás bits se pondrán a 0 ya que no se usarán

TCCR0B = 0b00000011; //PWM modo rapido, prescaler de 64

Código en C y simulación en Proteus

copiar código
#define F_CPU 16000000UL // Frecuencia de trabajo del microcontrolador
#include <avr/io.h>
#include <util/delay.h>

int main(void) {
DDRD = (1 << 6); // pin OC0A como salida
TCCR0A = 0b10000011; //PWM por el pin OC0A, no invertido, PWM modo rápido
TCCR0B = 0b00000011; //PWM modo rapido, prescaler de 64
OCR0A = 0; //inicio del conteo

while (1) {
OCR0A += 1; //el ancho de pulso se incrementa en una unidad cada 100 ms
_delay_ms(100);
}
}

Ejemplo 3: Manipulación del ancho de pulso

En este ejemplo se generará una señal PWM con un ancho de pulso variable desde el 0% hasta el 100% por el pin OC0A en forma no invertida en pasos de 25%, es el mismo ejemplo que el caso anterior pero esta vez se utilizará un pulsador conectado al pin PB5 para incrementar el valor del ancho de pulso y un pulsador conectado al pin PB4 para disminuirlo.

Fuente: Elaboración propia

Código en C y simulación en Proteus

copiar código
#define F_CPU 16000000UL // Frecuencia de trabajo del microcontrolador
#include <avr/io.h>
#include <util/delay.h>

int main(void) {
DDRD = (1 << 6); // pin OC0A como salida
TCCR0A = 0b10000011; //PWM por el pin OC0A, no invertido, PWM modo rapido
TCCR0B = 0b00000011; //PWM modo rapido, prescaler de 64
OCR0A = 0; //inicio del conteo

while (1) {
if (!(PINB & (1 << PORTB5))){ //si se presiona el pulsador
_delay_ms(150); // retardo antirebote
OCR0A += 64; //el ancho de pulso se incrementa un 25%
_delay_ms(100);
}
if (!(PINB & (1 << PORTB4))){ //si se presiona el pulsador
_delay_ms(150); // retardo antirebote
OCR0A -= 64; //el ancho de pulso se decrementa un 25%
_delay_ms(100);
}
}
}
