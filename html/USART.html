Comunicación Serial USART

¿Qué es la Comunicación Serial?

La comunicación serial se usa para la comunicación entre una placa Arduino y una computadora u otros dispositivos e inclusive otra placa Arduino. Todas las placas Arduino tienen al menos un puerto serie (también conocido como UART o USART)

Esta comunicación se logra a través de los pines digitales 0 (RX) y 1 (TX) (del puerto D) con la computadora a través de USB o con otros dispositivos que contengan estos mismos pines de comunicación. Por lo tanto, si se esta utilizando estas funciones, no se pueden usar también los pines 0 y 1 para entrada o salida digital.

Varios dispositivos como GPS, GSM, RFID, sensores, etc. necesitan comunicarse con el microcontrolador para transmitir o recibir información. Para comunicarse con el microcontrolador, se utilizan varios protocolos de comunicación como RS232, SPI, I2C, etc. Básicamente, un protocolo es un conjunto de reglas acordadas tanto por el remitente como por el receptor, en: ¿Cómo se empaquetan los datos? ¿Por cuántos bits están constituido los datos? ¿Cuándo comienzan y terminan los datos?

La comunicación serial en los pines TX / RX usa niveles lógicos TTL (5V o 3.3V dependiendo de la placa)

Protocolo de comunicación USART

El receptor y transmisor en serie síncrono y asíncrono universal (USART por sus siglas en inglés) es un protocolo de comunicación en serie que se utiliza para transmitir / recibir datos en serie a una velocidad de transmisión específica.

Con la ayuda de la USART, podemos enviar / recibir datos a una computadora u otros dispositivos. USART también se utiliza para interconectar el microcontrolador con varios módulos como Wi-Fi, Bluetooth, GPS, GSM, etc.

Cuenta con una operación Full duplex (registros de transmisión y recepción en serie de forma independiente)

Comunicación Asíncrona

La operación asincrónica significa que un proceso opera independientemente de otros procesos. La comunicación asíncrona significa que cada carácter (byte de datos) se coloca entre los bits de inicio y parada. El bit de inicio (para comenzar la comunicación entre transmisor-receptor) siempre es 0 (bajo) y el bit de parada (para finalizar la comunicación o para indicar que se terminó de transmitir/recibir todos los datos) siempre es 1 (alto)

Tasa de bits y tasa de baudios

La tasa de transferencia de datos en la comunicación de datos en serie se expresa en bps (bits por segundo). Otra terminología ampliamente utilizada para bps es la velocidad en baudios; significa, número de cambios en la señal por segundo. Aquí la señal está en bits, por lo tanto, tasa de bits = tasa de baudios.

Los principales componentes de hardware de USART son el generador de reloj, el transmisor y el receptor

La lógica de generación de reloj consta de una lógica de sincronización para la entrada de reloj externa utilizada por la operación esclava síncrona y el generador de velocidad en baudios.

Transmisor y receptor

El transmisor consta de un solo búfer de escritura, un registro de desplazamiento en serie, un generador de paridad y una lógica de control para manejar diferentes formatos de tramas en serie. El búfer de escritura permite una transferencia continua de datos sin demora entre cuadros.

El receptor es la parte más compleja del módulo USART debido a su reloj y unidades de recuperación de datos. Las unidades de recuperación se utilizan para la recepción de datos asíncrona. Además de las unidades de recuperación, el receptor incluye un verificador de paridad, lógica de control, un registro de cambio y un búfer de recepción de dos niveles (UDRn). El receptor admite los mismos formatos de trama que el transmisor y puede detectar errores de trama, desbordamiento de datos y errores de paridad.

Cada una de las unidades de hardware debe configurarse escribiendo bits en sus respectivos registros de control. El USART admite cuatro modos de funcionamiento: modo asíncrono normal, asíncrono de doble velocidad, síncrono maestro y síncrono esclavo.

Programación de la USART

Requerimientos

Los tres componentes de hardware principales que deben inicializarse antes de cualquier comunicación son el generador de reloj, el transmisor y el receptor. El proceso de inicialización normalmente consiste en configurar la velocidad en baudios, configurar el formato del marco (frame) y habilitar el transmisor o el receptor

El registro de velocidad en baudios USART (UBRRn) y el contador regresivo conectado a él funcionan como un pre escalador programable o generador de velocidad en baudios. El contador regresivo, que se ejecuta en el reloj del sistema (𝑓_𝑜𝑠𝑐), se carga con el valor UBRRn cada vez que el contador ha realizado una cuenta regresiva hasta cero o cuando se escribe en el registro UBRRnL. Se genera un reloj cada vez que el contador llega a cero. Este reloj es la salida del reloj del generador de velocidad en baudios (= fosc/(UBRRn + 1)). El transmisor divide la velocidad en baudios del generador en 2, 8 o 16 según el modo. La salida del generador de velocidad en baudios es utilizada directamente por el receptor. Sin embargo, las unidades de recuperación usan una máquina de estado que usa 2, 8 o 16 estados dependiendo del modo establecido por el estado de los bits UMSELn, U2Xn y DDR_XCKn.

Velocidad en Baudios

La velocidad en baudios se genera a partir del reloj del sistema con la ayuda del Prescaler y el circuito de reloj. El registro de velocidad en baudios de USART (UBRR0) controla el contador regresivo / preescalador programable para generar una señal de reloj en particular. El contador regresivo, que se ejecuta en el reloj del sistema (fosc), se carga con el valor UBRR0 cada vez que el contador llega a cero y genera un pulso de reloj.

Fuente: Microcontrollers ATmega328P Datasheet

La fórmula anterior se utiliza para calcular el valor correcto de UBBR0. Para Arduino UNO, el reloj del sistema funciona a 16 Mhz. Si pretendemos comunicarnos a una velocidad de 9600bps. El valor de UBBR0 debe ser UBBR0 = ((16,000,000 / 16 * 9600) -1) = 103 (redondeado)

Definición de la trama (Frame)

Una trama en serie se define como un carácter de bits de datos con bits de sincronización (bits de inicio y parada) y, opcionalmente, un bit de paridad para la verificación de errores. El USART acepta las 30 combinaciones de los siguientes como formatos de trama válidos:

  • 1 bit de inicio (siempre en 0)
  • 5, 6, 7, 8 o 9 bits de datos
  • sin bit de paridad, parida par o paridad impar
  • 1 o 2 bits de parada (siempre en 1)

Fuente: Microcontrollers ATmega328P Datasheet

Una trama comienza con el bit de inicio seguido del bit de datos menos significativo. Luego, los siguientes bits de datos, hasta un total de nueve, son correctos y terminan con el bit más significativo. Si está habilitado, el bit de paridad se inserta después de los bits de datos, antes de los bits de parada. Cuando se transmite una trama completa, puede ser seguida directamente por una nueva trama, o la línea de comunicación puede establecerse en un estado inactivo (alto).

Paridad

Estos bits habilitan y establecen el tipo de generación y verificación de paridad. Si está habilitado, el transmisor generará y enviará automáticamente la paridad de los bits de datos transmitidos dentro de cada trama. El receptor generará un valor de paridad para los datos entrantes y lo comparará con la configuración de UPMn. Si se detecta una discrepancia, se establecerá la bandera UPEn en UCSRnA.

Bits de datos

El formato de trama utilizado por USART lo establecen los bits UCSZn 2:0, UPMn1:0 y USBSn en UCSRnB y UCSRnC.

Los bits UCSZn1:0 combinados con el bit UCSZn2 en UCSRnB establecen el número de bits de datos (tamaño de carácter) en una trama que utilizan el receptor y el transmisor

Ejemplo 1: Transmisión de sondeo

Para este ejemplo se enviara carácter por carácter la palabra “Comunicación” a través del puerto serial cada un segundo, esto para entender más a detalle la configuración de los registros y el funcionamiento del módulo, para la simulación el circuito estará conectado al terminal virtual VSM que posee Proteus

El terminal virtual VSM le permite usar el teclado y la pantalla de la PC para enviar y recibir datos seriales asíncronos RS232 hacia y desde un sistema de microprocesador simulado. Es especialmente útil en la depuración, donde se puede usar para mostrar mensajes de depuración / seguimiento generados por el software que se está desarrollando.

Configuración inicial de parámetros

Definamos la frecuencia con la que trabajará el micro:
#define F_CPU 16000000UL //Frecuencia de trabajo
Modo de operación de la USART:
#define ASYN (0 << UMSEL00) // Selecciona el modo asincrono
Velocidad en baudios:
#define USART_BAUDRATE 9600 //Baudio de 9600
Pre-escalador:
#define BAUD_PRESCALER ((( F_CPU / ( USART_BAUDRATE * 16UL))) - 1) //formula dada en el DataSheet
Selección de los bit de paridad:
#define DISABLED (0 << UPM00) //Configuracion modo: Deshabilitado
#define EVEN_PARITY (2 << UPM00) //Configuracion modo: Paridad par
#define ODD_PARITY (3 << UPM00) //Configuracion modo: Paridad impar
#define PARITY_MODE DISABLED //Se selecciona el modo deshabilitado
Bits de parada:
#define ONE_BIT (0 << USBS0) //Se define Un bit de parada
#define TWO_BIT (1 << USBS0) //Se definen Dos bits de parada
#define STOP_BIT ONE_BIT //Se selecciona un bit de parada
Bits de datos:
#define FIVE_BIT (0 << UCSZ00) //Se definen 5 bits de datos
#define SIX_BIT (1 << UCSZ00) //Se definen 6 bits de datos
#define SEVEN_BIT (2 << UCSZ00) //Se definen 7 bits de datos
#define EIGHT_BIT (3 << UCSZ00) //Se definen 8 bits de datos
#define DATA_BIT EIGHT_BIT //Se selecciona, para el ejemplo, 8 bits de datos

Inicialización del módulo USART

void USART_Init() {
//Se configura el registro para ajustar los baudios
UBRR0H = BAUD_PRESCALER >> 8;
UBRR0L = BAUD_PRESCALER;
//Se configura la trama (Frame)
UCSR0C = ASYN | PARITY_MODE | STOP_BIT | DATA_BIT;
//Se habilita el transmisor y receptor
UCSR0B = (1 << RXEN0) | (1 << TXEN0);
}

void USART_TransmitPolling(uint8_t DataByte) {
while ((UCSR0A & (1 << UDRE0)) == 0) {}; //no hacer nada hasta que el UDRn este listo
UDR0 = DataByte; //Envía el nuevo dato
}

Ejemplo 2: Transmisión y Recepción de sondeo

La recepción de sondeo es el método de recepción más simple en el que el software de la aplicación sigue monitoreando el estado del hardware del receptor USART y lee los datos del búfer USART UDR0 solo cuando el hardware ha recibido un byte de datos

El siguiente ejemplo espera la entrada del usuario. Si la entrada en serie es "a" (ingresado por teclado), se enciende el LED conectado en el pin PB5 en la placa Arduino UNO. Para apagar el LED, se puede escribir cualquier otro carácter usando el Monitor Serie.

Fuente: Elaboración propia

Programa en C

copiar código
#define F_CPU 16000000UL //Frecuencia de trabajo

#include <avr/io.h>
#include <util/delay.h>

#define USART_BAUDRATE 9600 //Baudio de 9600
#define BAUD_PRESCALER ((( F_CPU / ( USART_BAUDRATE * 16UL))) - 1) //formula dada en el DataSheet
#define ASYN (0 << UMSEL00) // Selecciona el modo asincrono

#define DISABLED (0 << UPM00) //Configuracion modo: Deshabilitado
#define EVEN_PARITY (2 << UPM00) //Configuracion modo: Paridad par
#define ODD_PARITY (3 << UPM00) //Configuracion modo: Paridad impar
#define PARITY_MODE DISABLED //Se selecciona el modo deshabilitado

#define ONE_BIT (0 << USBS0) //Se define Un bit de parada
#define TWO_BIT (1 << USBS0) //Se definen Dos bits de parada
#define STOP_BIT ONE_BIT //Se selecciona un bit de parada

#define FIVE_BIT (0 << UCSZ00) //Se definen 5 bits de datos
#define SIX_BIT (1 << UCSZ00) //Se definen 6 bits de datos
#define SEVEN_BIT (2 << UCSZ00) //Se definen 7 bits de datos
#define EIGHT_BIT (3 << UCSZ00) //Se definen 8 bits de datos
#define DATA_BIT EIGHT_BIT //Se selecciona, para el ejemplo, 8 bits de datos

char word[] = {'C','a','r','a', 'c', 't', 'e', 'r', ':', ' '};
char LED[] = {'L','e','d',':', ' ', 'O', 'N'};
char word3[] = {'L','e','d',':', ' ', 'O', 'F', 'F'};

void USART_Init() {
//Se configura el registro para ajustar los baudios
UBRR0H = BAUD_PRESCALER >> 8;
UBRR0L = BAUD_PRESCALER;
//Se configura la trama (Frame)
UCSR0C = ASYN | PARITY_MODE | STOP_BIT | DATA_BIT;
//Se habilita el transmisor y receptor
UCSR0B = (1 << RXEN0) | (1 << TXEN0);
}

uint8_t USART_ReceivePolling() {
uint8_t DataByte;
while ((UCSR0A & (1 << RXC0)) == 0) {}; //no hacer nada hasta que se reciba el dato
DataByte = UDR0;
return DataByte;
}

void USART_TransmitPolling(uint8_t DataByte) {
while ((UCSR0A & (1 << UDRE0)) == 0) {}; //no hacer nada hasta que el UDRn este listo
UDR0 = DataByte; //Envía el nuevo dato
}

int main(void) {
DDRB |= 1 << 5; // PB5 como salida
USART_Init();
char LocalData;

while (1) {
for (int i = 0; i < 10; i++) {
USART_TransmitPolling(word[i]);
}
LocalData = USART_ReceivePolling();
USART_TransmitPolling(LocalData);
USART_TransmitPolling(13);

if (LocalData == 'a') {
for (int i = 0; i < 7; i++) {
USART_TransmitPolling(LED[i]);
}
PORTB |= 1 << 5;
}
else {
PORTB &= ~(1 << 5);
for (int i = 0; i < 8; i++) {
USART_TransmitPolling(word3[i]);
}
}

USART_TransmitPolling(13);
USART_TransmitPolling(13);
_delay_ms(500);
}
}
