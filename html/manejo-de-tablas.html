Manejo de tablas

Introducción

En ciertas ocasiones, se debe trabajar con tablas de datos en los sistemas digitales. Por ejemplo, se puede desear utilizar una tabla para relacionar el valor de una señal no lineal, o si se necesita un decodificador, como el BCD-7Segmentos (o cualquier otro particular), o se desea almacenar un conjunto de parámetros de ajuste de un determinado proceso.

Aquí se presentan al menos dos formas de trabajar con las tablas. Una de estas usa direccionamiento directo, e instrucciones que forman una subrutina con la estructura CASE (que en alto nivel no es más que una instrucción); y la otra forma es a través del uso del direccionamiento indirecto. Veremos las explicaciones con dos ejemplos.

Manejo de tablas con direccionamiento directo

El direccionamiento directo puede ser a registros o a memoria de datos. Recordemos que en este direccionamiento puede ser con uno o dos registros. Con DOS registros, los operandos están contenidos en el registro r (Rr) y d (Rd), y El resultado se almacena en el Registro d (Rd)

Ejemplo:

ADD 16, R17
MOV R0, R1
ADD R1, R2
SUB R1, R2

La estructura CASE consiste en comparar una variable (o en este caso, un registro), con varios casos de valores, de los cuales tomará solo uno a la vez, como vemos es la siguiente figura

Volviendo al direccionamiento directo, con registros de propósito general, e instrucciones de comparación, podemos construir la estructura CASE que conseguimos en los lenguajes de alto nivel, ya que ésta como tal no existe como instrucción en Assembler, Esta estructura puede elaborarse además con las instrucciones de salto, y el chequeo de las banderas, de la siguiente forma:

Algorítmo:

  • Carga valor en la variable (registro)
  • Compara con el valor1
  • Si es igual, salta a la etiqueta (llamada acciones1, en en ejemplo)
  • Compara con el valor2
  • Si es igual, salta a la etiqueta (llamada acciones1, en en ejemplo)
  • ….
  • Y asi sucesivamente.

Diseñe una ALU de 4 bits de datos utilizando un ATMega328P, en donde los operandos se lean del puerto B (nibble bajo para el operando A y nibble alto para el operando B), y el resultado se genere en el puerto D y con los 3 bits menos significativos del puerto C se defina la operación.

Las operaciones a realizar son:

Comencemos realizando el diagrama de flujo:

El programa en Assembler queda:

copiar código
.INCLUDE "M328PDEF.INC"
.DEF DATOA=R16
.DEF DATOB=R17
.DEF CODIGO=R18
.DEF RESULT=R21

.ORG 0 ; ALMACENARA EN LA LINE O DE PROGRAM MEMORY
.CSEG ; INICIO DEL SEGMENTO DE CODIGO DE PROGRAMA
RJMP INICIO

LDI R20, 255

OUT PORTB, R20 ; PULL UP EN PORTB
OUT PORTC, R20 ; PULL UP EN PORTC
OUT DDRD, R20 ; PORTD SALIDA

OUT DDRB, R19 ; PORTB ENTRADA
OUT DDRC, R19 ; PORTC ENTRADA

ANDI DATOB, $F0 ; DATOB ESTA EN LA PARTE ALTA. SE DEBE DESPLAZAR 4 POSICIONES A LA DERECHA
LSR DATOB
LSR DATOB
LSR DATOB
IN CODIGO, PINC ;LEE PORTC
ANDI CODIGO, $07 ; ENMASCARA LOS 3LSB

BREQ COD6

BREQ COD7
MOSTRAR:
OUT PORTD, RESULT

Manejo de tablas con direccionamiento indirecto a memoria de datos.

El direccionamiento puede ser indirecto a memoria de datos sencillo, o con desplazamiento (pre-decremento o con post-incremento).

Recordemos, en el direccionamiento indirecto de datos con post-incremento los registros X, Y o Z se incrementan después de la operación. La dirección del operando es el contenido del registro X-, Y- o Z- antes de incrementarse.

Ejemplo:
LD R16, Z+
ST Z+, R16

Ejemplo 2

Elabore un programa que incremente en una unidad, un contador de un digito, y muestre el valor en un display 7 segmentos. Resuelva el problema usando direccionamiento indirecto.

Inicialmente, construyamos la tabla de conversión para el display 7 segmentos. Donde, según la figura anterior, el segmento a se conecta a PB0, b se conecta a PB1, y así sucesivamente.

Seguidamente elaboremos el diagrama de flujo del programa principal y de las subrutinas.

La siguiente rutina permite representar un número en el display de 7 segmentos. Utiliza para ello el registro Z, que inicialmente apunta a la tabla de 7 segmentos. A este registro se le suma Contador, que es una variable entre 0 y 9 y, después, mediante acceso indirecto se carga el código 7 segmentos correspondiente en el puerto

copiar código
.INCLUDE "M328PDEF.INC"
.DEF TEMP = R16
/*Define un registro para uso temporal se ha utilizado el R16 para poder emplear los modos con direccionamiento indirecto*/
.DEF CONTADOR = R17 ;Cuenta el numero de pulsaciones
.DEF CERO = R18
.DSEG
.ORG $100
TABLA7SEG: .BYTE 10
/*Se reserva 10 bytes para una tabla de valores del convertidor bin7seg a partir de la 100h de ña SRAM*/
.CSEG
.ORG 0
RECALL INICIALIZA_PUERTOS ;Rutina que inicializa los puertos
RECALL INICIALIZA_TABLA7SEG ;Rutina que inicializa la tabla del convertidor
PRINCIPAL:
RECALL DISPLAY ;Representamos el valor de contador en el display
WAIT:
SBIC PIND, 0
RJMP WAIT
INC CONTADOR
CPI CONTADOR, 10
BREQ BORRA
RJMP ESPERA
BORRA:
LDI CONTADOR, 0

/*Este trozo de codigo esperamos a que se suelte sv1, si no se pone se incrementaria muchas veces el contador en el momento que se pulse*/
ESPERA:
SBIS PIND, 0
RJMP ESPERA
RJMP PRINCIPAL

INICIALIZA_PUERTOS:
;Aqui debe configurar PORTB como salida
LDI R19, 0b11111111
OUT DDRB, R19
;Aqui debe configurar PORTD como entrada
LDI R19, 0
OUT DDRD, R19
LDI TEMP, $FF
OUT PORTD, TEMP ;Se activan las resistencias Pull-Up del PORD
LDI CONTADOR, 0
LDI TEMP, 0
RET

/*La siguiente rutina inicializa la tabla 7 seg. Esta rutina seria innecesaria si se hubiera utilizado la memoria del programa para almacenarla*/
INICIALIZA_TABLA7SEG:
;Utilizamos Z para apuntar a la tabla. low() high() son directivas que devuelven el byte bajo o el alto de la direccion que se le pasa como argumento respectivamente
LDI ZH, HIGH(TABLA7SEG)
LDI ZL, LOW(TABLA7SEG)
LDI TEMP, 0b00111111 ;Codigo 7 segmentos del 0
ST Z+, TEMP
LDI TEMP, 0b00000110 ;Codigo 7 segmentos del 1
ST Z+, TEMP
LDI TEMP, 0b01011011 ;Codigo 7 segmentos del 2
ST Z+, TEMP
LDI TEMP, 0b01001111 ;Codigo 7 segmentos del 3
ST Z+, TEMP
LDI TEMP, 0b01100110 ;Codigo 7 segmentos del 4
ST Z+, TEMP
LDI TEMP, 0b01101101 ;Codigo 7 segmentos del 5
ST Z+, TEMP
LDI TEMP, 0b01111100 ;Codigo 7 segmentos del 6
ST Z+, TEMP
LDI TEMP, 0b00000111 ;Codigo 7 segmentos del 7
ST Z+, TEMP
LDI TEMP, 0b11111111 ;Codigo 7 segmentos del 8
ST Z+, TEMP
LDI TEMP, 0b01100111 ;Codigo 7 segmentos del 9
ST Z+, TEMP
LDI TEMP, 0b01100111
RET
DISPLAY:
LDI ZH, HIGH(TABLA7SEG)
LDI ZL, LOW(TABLA7SEG)
ADD ZL, CONTADOR ;el registro Z es de 16 bits, mientras que el contador es de 8
ADC ZH, CERO ;No olvidar sumar el acarreo que se genera del byte bajo a ZH
LD TEMP, Z
OUT PORTB, TEMP
RET
