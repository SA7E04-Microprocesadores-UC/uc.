Práctica #2: Manejo de tablas y display

Duración: 2 Semanas.

Objetivo General:

  • Eliminar el Rebote inherente a las entradas digitales implementadas mediante pulsadores y familiarizarse con el manejo de tablas.
Objetivos Específicos:
  • Realizar conversiones entre códigos mediante el manejo de tablas.
  • Manejar Displays siete segmentos.
  • Implementar rutinas de retardo de tiempo.
  • Eliminar el efecto rebote.

PRELABORATORIO:
  1. Programar la rutina vista en la clase de Teoría para realizar conversiones de Hexadecimal a 7 segmentos y manejar displays ánodo o cátodo común.
  2. ¿En qué consiste el fenómeno del “Rebote”? y cuál es el tiempo aproximado de duración del fenómeno de rebote en una entrada tipo pulsador digital?
  3. Programar una rutina de retardo por software para eliminar el rebote de pulsadores.
  4. Investigue el funcionamiento y conexiones para el modulo de Arduino KY-032: sensor de obstáculo.
  5. Investigue el funcionamiento y conexiones para el modulo de Arduino KY-004: pulsador
  6. Elabore el diagrama de flujo y programa en assembler para un proceso como el que se explica a continuación: Se tiene un proceso que en una de sus etapas traslada una a la vez, una serie de piezas terminadas, hasta una estación de encajonado. Una vez que se ha realizado el trasladado de la cantidad de piezas configuradas por un usuario, (previo al inicio del proceso); otra etapa del proceso retira la caja y coloca de forma automática, una caja vacía para repetir la operación. (ver Figura1)

Figura 1: Proceso de traslado de piezas

Inicialmente se ajusta la cantidad de piezas a detectar para trasladar a la estación de encajonado (solo se efectúa una vez, antes de iniciar el proceso), contando para esto con un pulsador C, el sistema debe ajustar la cantidad de piezas a trasladar. Cada vez que se presione y suelte la entrada C, un contador, se incrementará. Este número lo mostrará en un display 7 segmentos.

Una vez ajustada la cantidad de piezas, comienza el proceso activando el interruptor de inicio I.

  1. Si se detecta una pieza terminada (con un sensor P):, desciende el cilindro A, se activa automáticamente el vacío capaz de retener la pieza con la ventosa, se eleva la pieza, realiza un giro de 180°
  2. El cilindro A desciende, desactiva de forma automática el vacío que retiene la pieza con la ventosa, suelta la pieza, se retrae y el brazo manipulador retorna a su posición inicial.

El primer conjunto de movimientos (descenso del cilindro, tomar pieza, ascenso y giro de 180°) tiene una duración de 2 s; mientras que el segundo conjunto de movimientos (descenso, suelta la pieza, ascenso y giro) tiene una duración de 1 s. Finalizada la cantidad de detecciones configuradas. Encenderá un led durante 1 segundo y se reiniciará el proceso.

Se requiere que represente el proceso a través de un diagrama de flujo. Utilice la nomenclatura indicada en la figura 1 (P, C, A) en la elaboración del algoritmo, (esto es, no se requiere que indique el pin exacto del microcontrolador, donde se representaría cada uno de esos elementos)

Muestre a través de leds:
  • Un Led, el momento en el que el cilindro presente el primer conjunto de movimientos.
  • Un led, el momento en el que el cilindro presente el segundo conjunto de movimientos.
  • Un led que indica que llego a la pieza final
  • Un display 7 segmentos, la cantidad del conteo proveniente del pulsador C, y al momento de la detección de piezas, la cantidad de piezas detectadas por el sensor P.
LABORATORIO:

Para la implementación del algoritmo elaborado anteriormente deberá representar el botón para ajuste de la cantidad de piezas C mediante un pulsador (módulo KY-004) al pin PC1, el botón de inicio I se representará mediante un interruptor conectado en el pin PC0, la señal de la pieza detectada se representará a través del sensor KY-32. La señal proveniente del primer conjunto de movimientos (TRASLADA PIEZA) se representará mediante un led conectado en el pin PB1, la señal del cilindro cuando presente el segundo conjunto de movimientos (SUELTA PIEZA Y REGRESA) se representará mediante un led conectado en el pin PB2. El led para fin de conteo de pieza se representará en PB3, El display 7 segmentos representará la cantidad de piezas detectadas por el sensor P, y estará ubicado desde PD2 (el segmento a) hasta PD7 (segmento f), y PB0 (segmento g).

Actividades a desarrollar:
  • Muestre el prelaboratorio a su profesor(a), quien le interrogará sobre el mismo.
  • Codifique su programa en Assembler, dentro del entorno AVR Studio y ensamble su código hasta que no haya errores.
  • Efectúe la simulación de la aplicación en PROTEUS y verifique el funcionamiento con su profesor(a).
  • Programe el sistema de desarrollo Arduino UNO con la aplicación revisada mediante PROTEUS y verifique su funcionamiento con su profesor(a).
  • Elimine El rebote en el pulsador C. Vuelva a programar la tarjeta ARDUINO UNO y muestre a su profesor
Rutina de 3 Bucles para aproximadamente 1 segundo de retardo
copiar código
DELAY:
LDI R23, 0x42 ; R0 = 128
delay3:
LDI R22, 0xFF ; R1 = 255
delay2:
LDI R21, 0xFF ; R3 = 255
delay1:
DEC R21 ; Decrementa R21 en 1
CPI R21, 0
BRNE delay1 ; si R21 no es 0 entonces salta a delay2
DEC R22 ; Decrementa R22
CPI R22, 0
BRNE delay2 ; si R22 no es 0 entonces salta a delay1
DEC R23 ; Decrementa R23
CPI R23, 0
BRNE delay3 ; si R23 no es 0 entonces salta a delay1
RET
Rutina para mostrar en display 7 seg cátodo común el arduino uno.

Suponiendo que usa la tabla vista en clase

copiar código
MOSTRAR:
LDI ZH, HIGH(TABLA) ; R0 = 128
LDI ZL, LOW(TABLA) ; R1 = 255
ADD ZL, CONT ; CONT = el valor que trae del contador correspondiente al numero de pieza detectada por P ó el número de veces que se ha presionado el pulsador C de configuración inicial de conteo de piezas
LDI NUM, Z ; NUM = Registro temporal
CLR R18 ; R18 registro para manipulación del segmento g del display
ROL NUM ; rota y el MSB va al carry del registro SREG
ROL NUM ; rota nuevamente y el MSB va al carry del registro SREG (valor a enviar al display ya rotado 2 posiciones a la izquierda, para no usar PD1 ni PD0)
ROL R18 ; El valor del carry (el segmento g) pasa a r18
OUT PORTB, R18 ; se envía a PORTB
OUT PORTD, NUM ; el valor rotado de NUM pasa al PORTD
RET

LDI ZL, LOW(TABLA) ; R1 = 255

LDI R16, $5B ; 2

ST Z+, R16

ST Z+, R16
