Práctica #3: Interrupciones y temporizaciones

Duración: 2 Semanas.

Objetivos:

  • Desarrollar rutinas para el manejo de interrupciones.
  • Utilizar los Timers para producir retardos

PRELABORATORIO:
  • Investigue acerca de los registros y formulas requeridos para el uso de las interrupciones generados por los Timers del microcontrolador en estudio
  • ¿Cuánto es el tiempo máximo de desbordamiento que se puede obtener en el TIMER0 si la frecuencia del reloj es de 16 MHz?
  • Investigue como se pueden usar las interrupciones por desbordamiento del TIMER0 para ser utilizadas como base de tiempo para generar retardos superiores al valor máximo obtenido en el apartado anterior.
  • Investigue como trabajar con más de un display cuando se cuenta con puertos de E/S muy limitados
  • Investigue acerca de los registros requeridos para el uso de las interrupciones externas por cambio de flanco.
  • Investigue acerca del sensor de metal táctil KY-036.
LABORATORIO:

Se desea implementar un sistema de seguridad en una bóveda, que se active mediante un sensor de metal táctil estratégicamente instalado, el cual, al detectar metal, activará una señal luminosa (led) y arrancará un conteo de X segundos (pregunte a su profesor); una vez transcurrido éste tiempo (que será visualizado a través de dos displays cátodo común), se activará una alarma sonora que se desactivará sólo si se presiona el botón de STOP.

Como ya se mencionó, el conteo se visualizará en dos displays 7 segmentos cátodo común, D1 y D0 (ver figura)

  • Mientras la alarma sonora se encuentre activa, los displays se mantendrán apagados. En caso de no presionarse el pulsador de STOP, la alarma no se desactivará
  • Incluya un led que se active cuando el sistema esté encendido y a la espera de la activación del sensor.
  • Guíese por el circuito de la figura siguiente para la asignación de entradas y salidas.
CONSIDERACIONES:
  • Elabore una rutina basada en el uso del timer 0, para el conteo de tiempo en segundos.
  • Use interrupción externa INT0 para atender la activación del sensor de metal táctil KY-036.
  • Use interrupción externa INT1 para el botón de STOP.
  • El Buzzer se activa con un uno lógico, al igual que los leds.
Manejo de displays:

El siguiente programa toma los valores fijos de D0, para mostrarlos en el display (en un solo display). Usa la librería delay para generar un retardo por software en milisegundos o microsegundos

copiar código
#define F_CPU 16000000UL // Frecuencia de trabajo del microcontrolador
#include <avr/io.h>
#include <util/delay.h>

/***************** Declaracion de variables ******************************/
unsigned char Dig[10] = {0x3F,0x06,0x5B,0x4F,0x66,0x6D,0x7D,0x07,0x7F,0x6F};
uint8_t tempo, D0 = 0; // tempo se usara como variable temporal
/*************************************************************************/

/***************** Configuracion *************************/
void configuracion() {
DDRD = 0xF0; // Puerto D para usar el segmento g
DDRB = 0xFF; // PORTB como salida
PORTB = 0; // Segmentos del display apagados
PORTD = 0; // Segmentos del display apagados
}
/*********************************************************/

/********** funcion para mostrar en el display *************/
int mostrar() {
tempo = Dig[D0] & 0b01000000; // enmascara el segmento g, con esto se logra determinar cuando es necesario utilizar el segmento g
if (tempo == 0x40) PORTD = (1 << PORTD6); // enciende el segmento g cuando este sale en el arreglo
else PORTD = (0 << PORTD6); // de lo contrario este permanecera apagado
PORTB = Dig[D0]; // se muestran los dígitos correspondientes en el Display a través del puerto B y con el segmento g ubicado en el bit 6 del puerto D
_delay_ms(10);
}
/*********************************************************/

/************** Programa principal **********************/
int main(void) {
configuracion();
while (1) {
mostrar(); // Llama a la funcion para mostrar los digitos en el Display
D0 += 1; // D0 = D0 + 1; D0 se utiliza para recorrer la tabla
_delay_ms(300);
if (D0 >= 10) D0 = 0; // Si se recorrio toda la tabla D0 se reinicia a 0 para un nuevo comienzo
}
}
